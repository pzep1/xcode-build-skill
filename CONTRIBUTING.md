# Contributing to xcode-build-skill

Thank you for your interest in contributing to the Xcode Build Skill! This skill empowers Claude to build and manage iOS/macOS apps using native tools.

## How to Contribute

### 1. Fork and Clone
Fork the repository to your own GitHub account and clone it locally.

### 2. Create a Branch
Create a new branch for your feature or bug fix:
```bash
git checkout -b feature/amazing-feature
```

### 3. Make Changes
- Follow the existing code style and structure.
- If adding new skill capabilities, update `skills/xcode-build/SKILL.md` and `skills/xcode-build/CLI_REFERENCE.md`.
- If modifying UI automation guides, update `skills/xcode-build/XCUITEST_GUIDE.md`.

### 4. Test
Since this is a Claude skill, testing involves verifying the instructions are accurate and that the commands provided in the skill documentation work as expected with `xcodebuild` and `xcrun simctl`.

### 5. Submit a Pull Request
- Push your changes to your fork.
- Open a Pull Request against the `main` branch.
- Fill out the PR template completely.

## Code of Conduct
Please be respectful and professional in all interactions.

## Reporting Issues
If you encounter bugs or have feature requests, please use the Issue Templates provided in the `.github/ISSUE_TEMPLATE` directory.
